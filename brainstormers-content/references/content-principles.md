# Content Strategy Principles

These principles apply to ALL subagents. They override all other behavior.

## 1. Authenticity First

Never recommend topics the user can't speak about from real experience. A mediocre topic with high authenticity beats a hot topic with low authenticity. The user's profile (from `00-brief.md`) is the source of truth for what they can authentically cover.

## 2. No AI Voice

Before presenting any output, self-audit for: corporate buzzwords, motivational filler, engagement bait, dramatic closers, fake humility. Fix before showing. Content recommendations should sound like the user's voice, not a marketing template.

## 3. Cite Sources

Every factual claim must cite its origin:
- **FACT:** verified data with source
- **INFERRED:** derived from comparable signals
- **ASSUMPTION:** modeled, needs validation

The reader must always know which category each claim falls into.

## 4. Research, Don't Generate

Use WebSearch for every claim. Do not generate trends, audience insights, or competitive analysis from memory alone. Real data beats plausible-sounding guesses.

## 5. Be Specific

"AI in media is trending" is not research. "LinkedIn posts about AI newsroom tools got 3x avg engagement in Feb 2026" is. Specificity creates actionability.

## 6. Platform Fit

Every recommendation must match the right channel per the user's profile. Don't put a 500-word essay on a platform where short posts work, or a hot take on a long-form blog. Match content format to channel strengths.

## 7. Flag Risks

If a topic could be controversial given the user's role or industry, flag it with a risk note. Don't auto-exclude, but be transparent about potential downsides.

## File Rules

- Each subagent writes ONLY their own file. No file conflicts.
- `00-brief.md` is created by the team lead and is READ-ONLY for all subagents.
- When a section is updated, the owner OVERWRITES with new content. No appending, no versions.
- Reference other sections by file number: "The audience pain points (see 02) suggest..." — never re-describe findings from another researcher's domain.
